# 智能问答功能实现计划

## 任务分析与目标设定

**用户任务：**
1.  将主页面中的“deepseek聊天”更名为“智能问答”。
2.  添加 Ragflow 和 Dify 接入的支持。

根据收集到的信息（包括对 `calendar_with_react/src/components/Sidebar.jsx`、`calendar_with_react/src/api/deepseek.js` 的读取，以及对 “deepseek”、“ragflow”、“dify” 的文件搜索），我将任务分解为以下几个主要目标：

### 目标 1：更名“deepseek聊天”为“智能问答”
*   修改侧边栏菜单项文本。
*   修改路由路径。
*   修改相关组件名称和引用。
*   修改相关样式文件。

### 目标 2：集成 Dify
*   根据 `docs/dify_integration_plan.md` 文件，实现后端 DifyApp 模型、序列化器和视图。
*   实现前端 DifyAppList 和 DifyAppViewer 组件。
*   配置 React 路由。
*   更新侧边栏导航。
*   创建相关样式文件。

### 目标 3：集成 Ragflow
*   设计 Ragflow 的后端模型和 API 接口（如果需要）。
*   设计 Ragflow 的前端组件和页面。
*   配置 React 路由。
*   更新侧边栏导航。
*   创建相关样式文件。

### 目标 4：统一智能问答入口
*   将 DeepSeek、Ragflow 和 Dify 集成到一个统一的“智能问答”入口下，允许用户切换不同的后端服务。
