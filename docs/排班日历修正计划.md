# 排班日历修正计划

## 问题

排班详情中，值班领导和值班人员显示为“未知人员”。

## 根源分析

1.  **数据获取**: `useScheduleCalendarData` hook 分别通过 `scheduleApi.getSchedules()` 和 `scheduleApi.getPersonnel()` 获取排班数据和人员列表。排班数据中只包含人员ID。
2.  **数据处理**: `ScheduleCalendar.jsx` 组件接收排班列表和人员列表。它遍历排班列表，并根据人员ID从人员列表中查找相应的人员信息，然后将这些信息扁平地存放在 `extendedProps` 中。
3.  **数据展示**: `EventModal.jsx` 组件期望从 `currentEvent.extendedProps.scheduleDetails` 对象中获取嵌套的 `leader` 和 `staff` 对象来展示人员姓名和联系方式。
4.  **数据结构不匹配**: `ScheduleCalendar.jsx` 生成的 `extendedProps` 数据结构与 `EventModal.jsx` 期望的 `scheduleDetails` 对象结构不匹配，导致无法正确渲染人员信息。

## 修复计划

为了解决这个问题，我们将分两步进行：

### 1. 修改 `calendar_with_react/src/components/Calendar/ScheduleCalendar.jsx`

我们将调整 `ScheduleCalendar.jsx` 中 `extendedProps` 的数据结构，使其与 `EventModal.jsx` 的期望相匹配。

-   **创建 `scheduleDetails` 对象**: 在 `React.useMemo` 钩子中，为每个排班事件的 `extendedProps` 创建一个 `scheduleDetails` 对象。
-   **填充人员信息**: 在 `scheduleDetails` 对象中，创建 `leader` 和 `staff` 子对象，分别包含从 `personnel` 列表中查找到的完整人员信息（例如, `id`, `name`, `contact`）。

**修改前:**

```javascript
extendedProps: {
  type: 'SCHEDULE',
  duty_person_id: schedule.duty_person,
  duty_leader_id: schedule.duty_leader,
  duty_person_name: dutyPerson ? dutyPerson.name : '未知',
  duty_leader_name: dutyLeader ? dutyLeader.name : '未知',
  duty_person_phone: dutyPerson ? dutyPerson.phone : '',
  duty_leader_phone: dutyLeader ? dutyLeader.phone : '',
},
```

**修改后:**

```javascript
extendedProps: {
  type: 'SCHEDULE',
  scheduleDetails: {
    leader: dutyLeader ? { name: dutyLeader.name, contact: dutyLeader.phone } : { name: '未知', contact: '' },
    staff: dutyPerson ? { name: dutyPerson.name, contact: dutyPerson.phone } : { name: '未知', contact: '' },
    // 其他排班相关信息
  }
},
```

### 2. (可选) 调整 `calendar_with_react/src/components/Calendar/EventModal.jsx`

如果 `EventModal.jsx` 仍然无法正确显示信息，我们将检查其数据访问方式，确保它正确地从 `currentEvent.extendedProps.scheduleDetails` 中读取数据。

这个计划将统一数据结构，确保排班详情能够正确显示所有人员信息。
