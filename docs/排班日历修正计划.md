# 排班日历修正计划

## 任务目标
修正排班日历中的内容，使其显示人员的排班信息，而不是当前的试验日历内容。

## 当前分析总结

*   **前端路由：**
    *   `/calendar` 和 `/trial-calendar` 路由指向 `TrialCalendarPage`。
    *   `/shift-calendar` 路由指向 `ShiftCalendarPage`。
*   **前端组件结构：**
    *   `TrialCalendarPage` 和 `ShiftCalendarPage` 都包装了 `CalendarPage` 组件，并传递了 `calendarType` prop。
    *   `CalendarPage` 组件目前硬编码只渲染 `TrialCalendarContainer`，导致试验日历内容被显示在排班日历中。
    *   `calendar_with_react/src/components/Calendar/` 目录下已存在 `ScheduleCalendar.jsx` 和 `ScheduleCalendarContainer.jsx`，这表明排班日历的基础组件已存在。
*   **后端数据：**
    *   `DRFForVue/events/models.py` 定义了 `Schedule` 模型，包含 `duty_date`、`duty_person`、`duty_leader` 等排班相关字段。
    *   `DRFForVue/events/serializers.py` 定义了 `ScheduleSerializer`，用于序列化 `Schedule` 模型数据。
    *   `DRFForVue/events/views.py` 中的 `ScheduleViewSet` 提供了 `by-date-range` API 接口，可以按日期范围查询排班数据。

## 问题症结
前端 `CalendarPage` 没有根据 `calendarType` prop 动态渲染 `TrialCalendarContainer` 或 `ScheduleCalendarContainer`。

## 解决方案思路

1.  **修改 `CalendarPage.jsx`：** 根据 `calendarType` prop 动态渲染 `TrialCalendarContainer` 或 `ScheduleCalendarContainer`。
2.  **确保 `ScheduleCalendarContainer` 能够获取排班数据：** `ScheduleCalendarContainer` 需要调用后端 `ScheduleViewSet` 的 `by-date-range` API 来获取排班数据。

## 详细计划

1.  **修改 `CalendarPage.jsx` 文件：**
    *   导入 `ScheduleCalendarContainer`。
    *   将 `CalendarPage` 修改为接收 `calendarType` prop。
    *   使用条件渲染，根据 `calendarType` 的值来渲染 `TrialCalendarContainer` 或 `ScheduleCalendarContainer`。

2.  **修改 `ShiftCalendarPage.jsx` 文件：**
    *   确保 `ShiftCalendarPage` 传递 `calendarType="shift"` 给 `CalendarPage`。

3.  **修改 `TrialCalendarPage.jsx` 文件：**
    *   确保 `TrialCalendarPage` 传递 `calendarType="trial"` 给 `CalendarPage`。

4.  **确认 `ScheduleCalendarContainer.jsx` 的数据获取逻辑：**
    *   读取 `calendar_with_react/src/components/Calendar/ScheduleCalendarContainer.jsx` 的内容，确认其是否已经实现了从后端获取排班数据的逻辑。如果未实现，则需要添加。

5.  **确认 `TrialCalendarContainer.jsx` 的数据获取逻辑：**
    *   读取 `calendar_with_react/src/components/Calendar/TrialCalendarContainer.jsx` 的内容，确认其是否已经实现了从后端获取试验日历数据的逻辑。
