# OmniDesk 三部分架构优化计划

> 日期：2026-05-11
> 状态：待审批
> 涉及范围：后端、前端、桌面端

---

## 一、背景与目标

OmniDesk 当前分为三个部分：
1. **后端**（Django 4.2）— 部署在 Ubuntu 服务器上
2. **前端**（React 18.3 + CRA）— 浏览器访问，目标 Win7 客户机
3. **桌面端**（PyQt6）— Win7 客户机桌面应用，提供消息提醒和快速访问

整个系统运行在**无法访问互联网的内网**中。本次优化的目标是：

- 确保三个部分功能完整、职责清晰
- 修复 Win7 兼容性问题
- 提升离线部署和运行体验
- 扩展桌面端为"消息中心"定位

---

## 二、当前问题分析

### 2.1 后端

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 无实时推送能力 | 高 | 桌面端 5 秒轮询，效率低且无法实时推送新通知 |
| 缺少通知创建 API | 高 | 通知 model 已存在，但各业务模块未自动创建通知记录 |
| 无 API 版本前缀 | 中 | 路径为 `/api/` 而非 `/api/v1/`，未来可能产生版本冲突 |
| 外部 AI 服务降级缺失 | 中 | Ollama/Dify 在内网可能不可用时缺少 graceful fallback |
| 通知查询接口缺失 | 中 | 前端有 notification feature，但后端缺少标准 CRUD 接口 |

### 2.2 前端

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 双 UI 库并存 | 中 | Ant Design 5 + MUI 同时使用，包体积大，内网加载慢 |
| CRA 构建工具过时 | 低 | react-scripts 5.0.1 构建慢，HMR 不如 Vite |
| 无桌面端下载入口 | 中 | 前端没有提供桌面客户端的下载/安装引导 |
| Win7 测试缺失 | 中 | 有 browserslist 配置但无 Playwright 回归验证 |
| 通知功能未与后端对接 | 高 | 前端 notification feature 已存在，但缺少与后端通知模型的对接 |

### 2.3 桌面端

| 问题 | 严重度 | 说明 |
|------|--------|------|
| PyQt6 不支持 Win7 | **严重** | Win7 无法运行 PyQt6 应用，直接导致桌面端不可用 |
| Python 版本未锁定 | 高 | requirements.txt 无版本约束，Win7 最高仅支持 Python 3.8 |
| 功能单一 | 中 | 仅显示排班/试验/会议室，无通用通知提醒 |
| 缺少一键打开前端 | 中 | 与"快速访问前端"目标不符 |
| 无离线缓存 | 中 | 断网后无任何可用数据 |
| 无系统级通知弹窗 | 中 | 新消息到达时无弹窗提醒 |
| 无连接状态指示 | 低 | 无法判断后端是否可达 |

## 四、技术风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| PyQt5 在 Win7 上的 DPI 缩放问题 | UI 模糊或错位 | 在 `main.py` 中设置 `Qt.AA_EnableHighDpiScaling` 并测试 |
| Python 3.8 的 TLS 1.2 支持 | requests 可能无法连接 HTTPS 后端 | 内网使用 HTTP 可规避；如需 HTTPS，Win7 需安装 KB3140245 |
| 前端包体积过大 | 内网加载缓慢 | 检查所有依赖是否本地打包，移除未使用的 Ant Design 组件 |
| 通知信号与业务逻辑耦合 | 创建通知失败可能影响主流程 | 使用 `transaction.on_commit()` 或 Celery 异步创建 |

## 六、验收标准

### Win7 兼容性
- [ ] PyQt5 桌面端在 Win7 上能正常安装和运行
- [ ] 前端页面在 Chrome 109 / Edge 109 / Firefox 115 ESR 上渲染正常
- [ ] 无 ES2022+ 语法或 CSS `:has()` 等不兼容特性

### 通知系统
- [ ] 后端通知 CRUD API 可正常调用
- [ ] 业务事件（排班变更、公告发布）能自动创建通知
- [ ] 前端通知铃铛显示未读数量
- [ ] 桌面端通知 Tab 能拉取和展示通知

### 桌面端
- [ ] PyInstaller 打包为单文件 `.exe`
- [ ] 系统托盘右键菜单可用
- [ ] 一键打开浏览器前端页面
- [ ] 连接状态指示器正常工作
- [ ] 断网时显示缓存数据

### 部署
- [ ] 离线部署脚本包含桌面端安装步骤
- [ ] 前端提供桌面端下载入口
- [ ] 所有 Docker 镜像可导出为 `.tar`

## 七、不涉及的范围

- 不迁移前端从 CRA 到 Vite（风险大，单独计划）
- 不引入 Django Channels / WebSocket（当前轮询方案可满足需求，SSE 作为后续优化）
- 不添加 API 版本前缀 `/api/v1/`（改动范围大，单独计划）
- 不合并 Ant Design 和 MUI（渐进式优化，本次只保证功能正确）
