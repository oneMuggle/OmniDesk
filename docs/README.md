# OmniDesk Documentation

## Directory Structure

### `plans/` - Incomplete or ongoing plans

Design documents and implementation plans for features that are not yet fully implemented or still in progress.

- `sensor_management_plan.md` - Sensor management page implementation plan
- `schedule_refactoring_plan.md` - Schedule module refactoring plan
- `dify_app_fix_plan.md` - Dify app fix plan
- `csrf_fix_plan.md` - Django admin CSRF fix plan
- `backend_build_optimization_plan.md` - Backend build optimization plan
- `database_access_plan.md` - PostgreSQL external access plan
- `personnel_module_design.md` - Personnel module design report
- `win7_browser_compatibility_plan.md` - Windows 7 browser compatibility plan

### `technical/` - Completed technical documentation

Technical reference docs for implemented features and systems.

- `api_reference.md` - API reference documentation
- `deployment_manual.md` - Deployment manual (Ubuntu/Linux)
- `testing_strategy.md` - Testing strategy and coverage guide
- `cicd_guide.md` - CI/CD pipeline guide

### `user-manual/` - User-facing documentation

End-user guides and deployment instructions.

- `railway_deployment.md` - Railway deployment guide
- `deployment_guide.md` - Legacy Docker deployment guide (reference)
