# 会议室预约功能开发计划

## 概述
本计划旨在为“会议室预约”功能提供详细的开发步骤，涵盖后端API开发和前端页面实现。此功能将允许用户预约会议室，并提供管理员/经理对会议室和预约的管理功能。

## 需求回顾
- 用户可点击日历根据时间预约会议室。
- 预约时间只能在当前时间之后。
- 预约信息不能与已有预约信息重复。
- 用户可管理自己发布的预约（查看、编辑、删除）。
- 管理员和经理可以管理所有人的预约（查看、编辑、删除）。
- 预约需要记录预约主题（必填）、参与人员（可选）、会议描述（可选）。
- 会议室是固定的，但需要从后端动态获取列表。
- 管理员和经理需要管理会议室本身的信息（添加、编辑、删除会议室）。
- 管理员和经理需要设置会议室的可用时间段或维护时间。
- 管理员和经理需要查看会议室使用统计报告。

## 详细实施计划

### I. 后端开发

1.  **创建新的 Django 应用 `meeting_rooms`**:
    *   在 `omni_desk_backend` 目录下创建新的 Django 应用：`meeting_rooms`。
    *   将 `meeting_rooms` 添加到 `omni_desk_backend/omni_desk_backend/settings.py` 的 `INSTALLED_APPS` 中。

2.  **定义模型 (`omni_desk_backend/meeting_rooms/models.py`)**:
    *   `MeetingRoom` 模型：
        *   `name` (CharField, unique=True): 会议室名称，确保唯一性。
        *   `description` (TextField, blank=True, null=True): 描述。
        *   `capacity` (IntegerField, blank=True, null=True): 容量。
        *   `location` (CharField, blank=True, null=True): 位置。
    *   `MeetingRoomBooking` 模型：
        *   `meeting_room` (ForeignKey to `MeetingRoom`): 关联的会议室。
        *   `user` (ForeignKey to `CustomUser`): 预约用户。
        *   `start_time` (DateTimeField): 预约开始时间。
        *   `end_time` (DateTimeField): 预约结束时间。
        *   `title` (CharField): 预约主题 (必填)。
        *   `participants` (TextField, blank=True, null=True): 参与人员 (可选)。
        *   `description` (TextField, blank=True, null=True): 预约描述 (可选)。
        *   添加 `clean()` 方法或自定义验证器，以确保 `start_time` 在 `end_time` 之前，且预约不与现有预约和维护时间冲突。
    *   `MeetingRoomMaintenance` 模型：
        *   `meeting_room` (ForeignKey to `MeetingRoom`): 关联的会议室。
        *   `start_time` (DateTimeField): 维护开始时间。
        *   `end_time` (DateTimeField): 维护结束时间。
        *   `reason` (CharField): 维护原因。

3.  **定义序列化器 (`omni_desk_backend/meeting_rooms/serializers.py`)**:
    *   为 `MeetingRoom`、`MeetingRoomBooking` 和 `MeetingRoomMaintenance` 模型创建 `ModelSerializer`。

4.  **定义视图 (`omni_desk_backend/meeting_rooms/views.py`)**:
    *   `MeetingRoomViewSet` (ModelViewSet): 提供会议室的 CRUD API。
        *   添加权限控制，确保只有管理员和经理可以进行 CRUD 操作。
    *   `MeetingRoomBookingViewSet` (ModelViewSet): 提供预约的 CRUD API。
        *   自定义 `perform_create` 和 `perform_update` 方法，实现预约时间冲突检测（包括与现有预约和维护时间的冲突）。
        *   自定义 `get_queryset` 方法，确保普通用户只能看到自己的预约，而管理员/经理可以看到所有预约。
        *   使用 Django REST Framework 的权限类进行权限控制。
    *   `MeetingRoomMaintenanceViewSet` (ModelViewSet):
        *   提供会议室维护时间段的 CRUD API。
        *   添加权限控制，确保只有管理员和经理可以进行 CRUD 操作。
    *   `MeetingRoomStatsAPIView` (APIView):
        *   提供会议室使用统计报告的 API。
        *   可接受参数（如时间范围、会议室ID），返回统计数据（如总预约时长、预约次数等）。
        *   添加权限控制，确保只有管理员和经理可以访问。

5.  **定义 URL 路由 (`omni_desk_backend/meeting_rooms/urls.py`)**:
    *   使用 `DefaultRouter` 为 `MeetingRoomViewSet`、`MeetingRoomBookingViewSet` 和 `MeetingRoomMaintenanceViewSet` 注册路由。
    *   为 `MeetingRoomStatsAPIView` 添加单独的 URL 路径。

6.  **在主 URL 中包含应用路由 (`omni_desk_backend/omni_desk_backend/urls.py`)**:
    *   在 `urlpatterns` 中添加 `path('api/', include('meeting_rooms.urls'))`。

7.  **数据库迁移**:
    *   运行 `python manage.py makemigrations meeting_rooms`。
    *   运行 `python manage.py migrate`。

### II. 前端开发

1.  **创建 API 服务文件 (`omni_desk_frontend/src/api/meetingRoomApi.js`)**:
    *   封装对后端 `meeting_rooms` API 的请求，包括获取会议室列表、创建/获取/更新/删除预约、管理维护时间、获取统计数据等。

2.  **创建会议室预约页面组件 (`omni_desk_frontend/src/pages/MeetingRoomBookingPage.jsx`)**:
    *   导入 `react-big-calendar` 或其他日历库。
    *   获取会议室列表（通过 `meetingRoomApi.getMeetingRooms()`）。
    *   使用数据获取 Hook，获取预约数据。
    *   展示日历视图，用户可点击日期或时间段进行预约。
    *   实现预约表单，包含会议室选择、开始/结束时间、主题（必填）、参与人员（可选）、描述（可选）等字段。
    *   展示用户的预约列表。
    *   根据用户角色，判断是否展示所有预约列表，以及是否允许编辑/删除所有预约。
    *   实现预约的编辑和删除功能。

3.  **新增会议室管理页面组件 (`omni_desk_frontend/src/pages/MeetingRoomManagementPage.jsx`)**:
    *   此页面仅限管理员和经理访问。
    *   展示会议室列表，包含添加、编辑、删除会议室的功能。
    *   提供界面来管理会议室的维护时间段（添加、编辑、删除）。
    *   展示会议室使用统计报告（可能通过图表展示）。

4.  **更新前端路由 (`omni_desk_frontend/src/routes/index.js`)**:
    *   导入 `MeetingRoomBookingPage` 和 `MeetingRoomManagementPage`。
    *   在 `path: "/"` 对应的 `children` 数组中添加：
        ```javascript
        {
            path: "meeting-rooms",
            element: <ProtectedRoute><MeetingRoomBookingPage /></ProtectedRoute>
        },
        ```
    *   在 `path: "/admin"` 对应的 `children` 数组中添加：
        ```javascript
        {
            path: "meeting-room-management",
            element: <ProtectedRoute roles={['admin', 'manager']}><MeetingRoomManagementPage /></ProtectedRoute>
        },
        ```

5.  **更新导航栏 (`omni_desk_frontend/src/components/Sidebar.jsx`)**:
    *   在 `menuItems` 数组中找到 `日历` 的 `subItems`，添加新的子项：
        ```javascript
        { to: "/meeting-rooms", text: "会议室预约", permission: null },
        ```
    *   在 `menuItems` 数组中找到 `管理中心` 的 `subItems`，添加新的子项：
        ```javascript
        { to: "/admin/meeting-room-management", text: "会议室管理", permission: ["admin", "manager"] },
        ```
