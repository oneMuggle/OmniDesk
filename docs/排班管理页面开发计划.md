# 排班管理页面开发计划

## 任务目标
在管理面板中，新增人员排班管理页面，允许管理员和经理管理排班信息。

## 核心功能
1.  **排班列表展示**: 显示现有排班数据，包括日期、值班人员和值班领导。
2.  **排班创建/编辑**: 允许创建新排班或修改现有排班，选择值班日期、值班人员和值班领导。
3.  **排班删除**: 允许删除现有排班。
4.  **人员管理**: 允许添加、编辑和删除值班人员信息。
5.  **权限控制**: 确保只有管理员和经理可以进行排班和人员的管理操作。
6.  **日历可视化**: 在排班管理页面中集成日历组件，用于可视化展示排班信息，并支持交互。
7.  **排班自动生成**: 允许管理员或经理根据指定的人员顺序和起始日期，自动生成排班信息。

## 技术栈
*   **前端**: React (结合现有 `calendar_with_react` 项目结构)，使用 Ant Design 组件库。
*   **后端**: Django REST Framework (结合现有 `DRFForVue` 项目结构)。

## 详细计划步骤

### 阶段一：后端增强 (DRFForVue)
1.  **创建 `Personnel` 管理接口**: 确保 `DRFForVue/events/views.py` 中的 `ResponsiblePersonViewSet` 完全支持 `Personnel` 模型的 CRUD 操作，并暴露正确的 API 端点 (`/events/personnel/`)。
2.  **新增批量排班生成 API**：
    *   在 `DRFForVue/events/views.py` 的 `ScheduleViewSet` 中新增一个 `action`，例如 `generate_schedules`。
    *   该 API 接收参数：
        *   `start_date`：起始日期。
        *   `personnel_order`：一个人员 ID 列表，表示排班顺序。
        *   `duration_days` (可选)：生成排班的天数，默认为一个月或一年。
    *   根据人员顺序和起始日期，计算并批量创建 `Schedule` 记录。
    *   需要考虑日期冲突处理，可以利用 `ScheduleSerializer` 已有的 `override` 机制。

### 阶段二：前端开发 (calendar_with_react)
1.  **新增排班管理页面组件 (`ScheduleManagementPage.jsx`)**:
    *   在 `calendar_with_react/src/pages/` 目录下创建 `ScheduleManagementPage.jsx`。
    *   **集成 Ant Design 日历组件**: 在页面中嵌入一个 Ant Design 的日历组件，用于展示排班信息。
    *   **日历事件显示**: 从后端获取排班数据 (`scheduleApi.getSchedules` 或 `by-date-range` API)，并将其映射到日历事件中。可以利用 `ScheduleCalendar.jsx` 中的逻辑来处理数据和显示。
    *   **排班列表展示**: 日历下方或侧边可以保留一个表格，用于详细展示排班列表，并提供“新增”、“编辑”、“删除”等操作按钮。
    *   **排班创建/编辑模态框 (`ScheduleFormModal.jsx`)**: 复用或创建模态框，用于创建和编辑单个排班。
    *   **人员选择器**: 在表单中使用 Ant Design 的 `Select` 组件，结合 `scheduleApi.getPersonnel()` 获取人员列表。
    *   **新增排班自动生成功能 UI**:
        *   在页面上添加一个区域或按钮，用于触发自动生成排班功能。
        *   包含：日期选择器（起始日期）、人员顺序输入（可拖拽排序的列表或多选框），以及生成天数输入框。
        *   点击生成按钮时，调用后端新增的批量排班生成 API。
2.  **创建人员管理页面组件 (`PersonnelManagementPage.jsx`)**:
    *   在 `calendar_with_react/src/pages/` 目录下创建 `PersonnelManagementPage.jsx`。
    *   使用 Ant Design 的 `Table` 组件显示人员列表。
    *   提供“新增”、“编辑”、“删除”人员信息的表单和操作。
    *   确保调用 `scheduleApi.getPersonnel()` 以及可能的 `createPersonnel`/`updatePersonnel`/`deletePersonnel` 方法（如果 `scheduleApi.js` 中没有，则需要新增）。
3.  **集成到管理面板导航**:
    *   修改 `calendar_with_react/src/components/Sidebar.jsx`，在侧边栏中添加“排班管理”和“人员管理”的导航链接。
    *   **权限控制**: 根据 `localStorage` 中的 `user.role` 判断用户权限，动态显示导航链接。

### 阶段三：路由配置 (calendar_with_react)
1.  **配置前端路由**:
    *   在 `calendar_with_react/src/App.jsx` 或相应的路由配置文件中，为 `ScheduleManagementPage` 和 `PersonnelManagementPage` 配置新的路由，例如 `/admin/schedules` 和 `/admin/personnel`。
    *   使用 `AdminLayout.jsx` 作为这些页面的布局组件。
    *   **权限路由**: 创建一个权限路由组件，确保只有管理员和经理可以访问这些路由，并在路由守卫中检查 `localStorage` 中的 `user.role`。

### 阶段四：权限管理
1.  **前端权限控制**:
    *   在 `Sidebar.jsx` 和权限路由组件中，根据 `localStorage` 中的 `user.role` 进行权限判断。
